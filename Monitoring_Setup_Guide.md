## üö® ALERT RESPONSE PROTOCOLS

### Severity Classification

**CRITICAL (Act within 1 hour)**
- Major fake announcement about company (merger, bankruptcy, etc.)
- Coordinated attack by multiple accounts
- Viral fake content (>1,000 shares)
- Content causing stock price movement
- Death threats or serious legal issues

**HIGH (Act within 4 hours)**
- Fake account with significant following (>1,000 followers)
- Defamatory blog post on major platform
- Unauthorized investment advice
- Fake job postings
- Trademark infringement on product/service

**MEDIUM (Act within 24 hours)**
- Small fake accounts
- Negative comments/reviews (legitimate grievance)
- Unauthorized use of logo/brand
- Forum discussions with misinformation

**LOW (Act within 1 week)**
- Minor mentions without impact
- Old content resurfacing
- Competitor comparisons (fair use)
- General industry discussions

---

### Response Workflows

#### CRITICAL ALERT WORKFLOW

**Hour 1:**
1. **Verify** (10 min)
   - Confirm it's actually fake/unauthorized
   - Assess scope and reach
   - Screenshot and archive

2. **Notify** (15 min)
   - Alert CEO/MD immediately
   - Brief legal team
   - Inform PR team
   - Activate crisis team

3. **Act** (30 min)
   - Post official statement on all channels
   - Report to platforms immediately
   - File cyber crime complaint (if applicable)
   - Contact SEBI (if stock manipulation)

4. **Monitor** (Ongoing)
   - Track spread
   - Identify all sources
   - Document everything

**Hours 2-4:**
- Legal team drafts cease and desist
- PR team prepares detailed response
- Contact major media to provide correct info
- Email all shareholders if needed

**Day 1-7:**
- Follow up on platform takedowns
- Legal action against perpetrators
- Media outreach
- Stakeholder communication
- Daily updates to leadership

---

#### HIGH ALERT WORKFLOW

**Hour 1-4:**
1. **Document** (1 hour)
   - Screenshots (full page, URL visible)
   - Archive (archive.org, archive.today)
   - WHOIS lookup (if website)
   - Log all details in monitoring dashboard

2. **Report** (1 hour)
   - Submit takedown request to platform
   - Fill out appropriate forms
   - Upload evidence

3. **Respond** (2 hours)
   - Post warning on official social media
   - Update website investor alert page
   - Notify key stakeholders if needed
   - Draft cease and desist (if contact info available)

**Day 2-7:**
- Follow up on reports
- Check if content removed
- Escalate if not removed
- Legal letter if contact information obtained

---

#### MEDIUM ALERT WORKFLOW

**Day 1:**
- Document thoroughly
- Log in system
- Assess if escalation needed

**Day 2:**
- Report to platform
- Prepare standard response

**Day 3-7:**
- Monitor for escalation
- Follow up on reports
- Update stakeholders if pattern emerges

### Scripts & Tools

#### Python Script: Automated Twitter Monitoring

```python
# Simple Twitter monitoring script
# Requires: tweepy library

import tweepy
import datetime

# Configuration (replace with your credentials)
API_KEY = "your_api_key"
API_SECRET = "your_api_secret"
ACCESS_TOKEN = "your_access_token"
ACCESS_SECRET = "your_access_secret"

# Authenticate
auth = tweepy.OAuthHandler(API_KEY, API_SECRET)
auth.set_access_token(ACCESS_TOKEN, ACCESS_SECRET)
api = tweepy.API(auth)

# Keywords to monitor
KEYWORDS = ["Nirman Agri Genetics", "NirmanAgri", "YOUR_STOCK_SYMBOL"]
NEGATIVE_KEYWORDS = ["scam", "fraud", "fake"]

# Search recent tweets
def monitor_tweets():
    for keyword in KEYWORDS:
        tweets = api.search_tweets(q=keyword, count=100, lang="en")
        
        for tweet in tweets:
            # Check for negative keywords
            tweet_text = tweet.text.lower()
            has_negative = any(neg in tweet_text for neg in NEGATIVE_KEYWORDS)
            
            if has_negative:
                # Alert logic here
                print(f"ALERT: Negative tweet found!")
                print(f"Tweet: {tweet.text}")
                print(f"User: @{tweet.user.screen_name}")
                print(f"URL: https://twitter.com/{tweet.user.screen_name}/status/{tweet.id}")
                print(f"Followers: {tweet.user.followers_count}")
                print("---")
                
                # You can add:
                # - Send email alert
                # - Add to Google Sheet
                # - Post to Slack
                # - Send SMS

# Run every hour
if __name__ == "__main__":
    monitor_tweets()
```

**Usage**:
1. Install Python
2. Install tweepy: `pip install tweepy`
3. Get Twitter API credentials from developer.twitter.com
4. Run script: `python twitter_monitor.py`
5. Schedule with cron (Linux) or Task Scheduler (Windows)
